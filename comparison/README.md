# Cross-rule architecture-controlled comparison

## In simple words

This folder compares all four learning rules implemented in this repository —
backpropagation, feedback alignment, predictive coding, and Hebbian/Oja — under
one fixed, controlled architecture, so differences in forgetting can be
attributed to the learning rule rather than to model capacity or
hyperparameters.

## Controlled protocol

- Architecture: `784 -> 1000 -> 10` (the widest single-hidden-layer setting
  used elsewhere in this repository).
- Batch size: 32.
- Learning rate: 0.001 for every rule.
- Optimizer: Adam for backprop and feedback alignment (the two rules that
  train via `loss.backward()`); no optimizer for predictive coding and Hebbian,
  which update their own weights directly (JAX/`jpc` and local Oja/delta
  updates respectively) — the shared harness (`src/experiments/forgetting.py`)
  resolves this automatically per `model_type`.
- Bias: false.
- Full MNIST, old classes `0-5`, new classes `6-9`, seed 0.
- Phase 1: 20 recorded epochs (1 baseline + 19 update epochs, matching the
  shared training convention). Phase 2: 20 recorded epochs, run once with only
  the new classes (sequential) and once with the full 0-9 loader
  (interleaved).

## Final result

| Method | Condition | Old before | Old after | Forgetting (relative) | New after |
|---|---|---:|---:|---:|---:|
| Backprop | Sequential | 99.05% | 4.08% | 95.9% | 99.16% |
| Backprop | Interleaved | 99.05% | 97.88% | 1.2% | 98.07% |
| Feedback Alignment | Sequential | 99.02% | 0.22% | 99.8% | 98.95% |
| Feedback Alignment | Interleaved | 99.02% | 97.94% | 1.1% | 96.98% |
| Predictive Coding | Sequential | 98.89% | 0.00% | 100.0% | 98.68% |
| Predictive Coding | Interleaved | 98.89% | 97.41% | 1.5% | 97.36% |
| Hebbian | Sequential | 90.15% | 0.00% | 100.0% | 91.78% |
| Hebbian | Interleaved | 90.15% | 84.44% | 6.3% | 84.48% |

"Forgetting (relative)" is normalized to each method's own phase-1 accuracy —
`(old_before - old_after) / old_before * 100` — rather than raw percentage
points, because Hebbian reaches a lower phase-1 accuracy than the other three
methods; normalizing removes that confound so the comparison is fair.

**Headline finding:** under sequential training, all four learning rules
collapse to essentially 0% retained accuracy on the old task (95.9-100%
relative forgetting) — biological plausibility neither helps nor hurts with
catastrophic forgetting under this protocol, and even plain backpropagation is
not immune. The interleaved control is where a real difference shows up:
Hebbian forgets proportionally more (6.3%) than the other three methods
(1.1-1.5%), a genuine ~4-6x gap that holds up after controlling for its lower
starting competence.

**Caveat:** Hebbian's lower phase-1 accuracy (90.15% vs. ~99% for the other
three) reflects a real difference in learning capacity for this local rule at
this architecture/learning-rate setting — it is a separate finding from the
forgetting comparison above, not an artifact of it.

## Files

- `results.json` — raw per-epoch traces and summary statistics for backprop,
  feedback alignment, and predictive coding, generated by `run_comparison.py`.
- `run_comparison.py` — regenerates `results.json` from scratch (requires the
  full `requirements.txt`, including the JAX/`jpc` stack for predictive
  coding; takes roughly 15-30 minutes on CPU).
- `make_plots.py` — builds the three PNGs below from `results.json` plus
  Hebbian's existing `../hebbian/FINAL (learning rate 0.001)/1000/results.json`
  (no JAX required, just matplotlib/numpy).
- `forgetting_curves.png` — old-class accuracy across all 40 epochs, sequential
  vs. interleaved, all four methods overlaid.
- `forgetting_magnitude_bars.png` — relative forgetting by method and
  condition.
- `comparison_table.png` — the table above, rendered as an image.

Hebbian's data is reused as-is from `hebbian/FINAL (learning rate 0.001)/1000/`
rather than duplicated here, since it was already generated under this exact
architecture/learning-rate setting.
